
## Git (базовые команды)

### Инициализация репозитория

```bash
git init
```

### Состояние файлов до коммита

```bash
git status
```

## Коммиты

### Добавить файл/файлы из папки в коммит

```bash
git add ПУТЬ
```

### Добавить все файлы из текущей папки в коммит

```bash
git add .
```

### Коммит/сохранение

```bash
git commit -m "ТЕКСТ"
```

### Полностью удалить последний коммит (все изменения в файлах коммита тоже удалятся)

```bash
git reset --hard HEAD~1
```

## Ветки

### Создание ветки

```bash
git branch ИМЯ-ВЕТКИ
```

### Перейти в ветку

```bash
git checkout ИМЯ-ВЕТКИ
```

### Создать и перейти в ветку

```bash
git checkout -b ИМЯ-ВЕТКИ
```

### Удалить ветку

```bash
git branch -D ИМЯ-ВЕТКИ
```

### Перенести в текущую ветку все изменения из введённой ветки

```bash
git merge ИМЯ-ВЕТКИ
```

> Обычно когда нужно из побочной ветки перенести в `main`

### Переименовать текущую ветку

```bash
git branch -M ИМЯ-ВЕТКИ
```

## Config

### Привязать почту к коммитам (обязательно)

```bash
git config --global user.email ЭЛЕКТРОННАЯ-ПОЧТА
```

### Установить отображаеммое при коммите имя (обязательно)

```bash
git config --global user.name ФАМИЛИЯ-ИМЯ
```

### Чтобы был только LF, а не CRLF

```bash
git config --global core.autocrlf false
```

### Изменение слияния (обычно не используют)

Устанавливает поведение git pull так, чтобы при получении изменений из удаленного репозитория всегда выполнялось **слияние (merge)**, а не перебазирование (rebase) — это настройка по умолчанию, которая явно говорит Git сливать ветки, создавая коммит слияния, и сохраняет историю в виде ветвей, как она есть.

```bash
git config pull.rebase false
```

## Получение изменений из удалённого репозитория

### Внести изменения из удалённого репозитория в локальный

```bash
git pull origin ИМЯ-ВЕТКИ
```

### Удалить ссылки веток, отсутствующих в удалённом репозитории

Удалить ССЫЛКИ на удаленные ветки, которые локально указаны, что есть, но по факту их уже нет в удалённом репозитории

```bash
git fetch --prune
```

### Удалить локальные ветки, отсутствующие в удалённом репозитории

Эта команда находит все локальные ветки, чьи удалённые аналоги удалены на сервере,и удаляет их локально (только если они безопасно удаляемы).

#### Посмотреть что будет удалено

```bash
git branch -vv | Select-String ': gone]' | ForEach-Object { $_ -replace '^\*?\s*', '' -replace '\s+.*$', '' }
```

#### Удалить

```bash
git branch -vv | Select-String ': gone]' | ForEach-Object { $branch = $_ -replace '^\*?\s*', '' -replace '\s+.*$', ''; git branch -d $branch }
```

> Если хочешь жёстко удалить такие ветки (даже не слитые в `main`) - надо поменять `-d` на `-D`.
